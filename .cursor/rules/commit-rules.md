# Commit Rules

**Reference**: See `.standards/git_workflow_standards.md` for complete Git workflow and commit standards.

## Quick Reference

### Conventional Commit Format

```bash
type(scope): message
```

### Allowed Types

- `feat` - New features
- `fix` - Bug fixes
- `docs` - Documentation changes
- `refactor` - Code refactoring
- `test` - Adding or updating tests
- `chore` - Maintenance tasks

### Format Rules

- Keep commit title under 72 characters
- Use conventional commit format: `type(scope): message`
- Separate title and body with blank line
- Put additional details in commit body

### Examples

```bash
feat(auth): add email verification functionality
fix(payment): correct tax calculation for international payments
docs: update API documentation for subscription endpoints
refactor(user): simplify user service implementation
test(auth): add tests for password reset flow
```
